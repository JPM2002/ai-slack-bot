# Engineering Onboarding Checklist

## Week 1: Setup & Orientation
- [ ] Get Slack access (channels: #engineering, #deployments, #incidents)
- [ ] Get GitHub org access + clone core repos
- [ ] Set up local dev environment (Node 20+, Docker, Supabase CLI)
- [ ] Meet your team lead and buddy
- [ ] Read the architecture overview doc

## Week 2: First Contribution
- [ ] Pick up a "good first issue" from Linear
- [ ] Complete your first PR with code review
- [ ] Join a sprint planning session
- [ ] Set up Amplitude access and explore key dashboards

## Week 3: Deeper Integration
- [ ] Shadow a deployment
- [ ] Attend a post-mortem or incident review
- [ ] Start working on your first feature ticket
- [ ] 1:1 with your manager to discuss 30-day goals

## Tools We Use
Linear (tickets), Amplitude (analytics), Notion (docs), Slack (comms), GitHub (code), n8n (automation), Supabase (DB), Figma (design)
